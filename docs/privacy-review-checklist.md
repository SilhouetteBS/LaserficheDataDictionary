# Privacy Review Checklist

Use this checklist before importing, publishing, or discussing a submitted
schema export.

## Submission Intake

- [ ] Submission came through an approved GitHub issue or maintainer-approved channel.
- [ ] Product key is one of `forms`, `lfds`, `repository`, or `workflow`.
- [ ] Product version is explicit and does not come from the SQL Server database name.
- [ ] Database role describes the Laserfiche product database role, not an environment name.
- [ ] Export files were generated by `docs/sql-server-schema-export.sql` or an approved equivalent.

## Required Files

- [ ] `manifest.json`
- [ ] `schemas.json`
- [ ] `tables.json`
- [ ] `columns.json`
- [ ] `primaryAndUniqueKeys.json`
- [ ] `foreignKeys.json`
- [ ] `indexes.json`
- [ ] `views.json`
- [ ] `routines.json`
- [ ] `triggers.json`
- [ ] `dependencies.json`

Empty optional files are acceptable only when the export script produced no rows.

## Sensitive Data Rejection Checks

Reject and regenerate the export if any file includes:

- [ ] table row data
- [ ] customer names or customer-specific values
- [ ] user record values
- [ ] document, folder, form submission, or workflow instance values
- [ ] SQL Server database names
- [ ] SQL Server server names
- [ ] SQL Server version, compatibility level, file paths, or instance configuration
- [ ] credentials, connection strings, tokens, API keys, certificates, or secrets
- [ ] screenshots of production records
- [ ] private notes or internal research artifacts

## Metadata Quality Checks

- [ ] `dbo.sysdiagrams` is absent or excluded by import.
- [ ] Product/version in every file agrees with `manifest.json`.
- [ ] Missing `views.json`, `routines.json`, `triggers.json`, or `dependencies.json` rows are explained.
- [ ] Duplicate product/version submissions are marked as replacements and reviewed intentionally.
- [ ] `npm run validate:data` passes after import.
- [ ] `npm run validate:notes` passes if notes were changed.

## Publication Checks

- [ ] Generated `schema.json` and manual `notes.json` remain separate.
- [ ] `public/data` and `data` copies match for the imported product/version.
- [ ] AI export package was regenerated when schema or notes changed.
- [ ] Public docs list the new product/version.
- [ ] Public build is generated without `VITE_ENABLE_EDITING=true`.
- [ ] Known limitations and read-only support warning remain visible.
