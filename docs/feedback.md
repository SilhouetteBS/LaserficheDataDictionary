# Feedback

This project uses GitHub Issues for feedback instead of analytics or in-app tracking.

Use issues for:

- Schema export submissions
- Documentation corrections
- Bug reports
- Public build or deployment problems

Do not include:

- Row data
- Customer-specific values
- Database names
- Server names
- Credentials
- Screenshots that show production records

Preferred templates:

- `Schema export submission`
- `Documentation correction`
- `Bug report`

For general discussion, open an issue and clearly mark it as a question or proposal.
